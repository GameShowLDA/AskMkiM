2. Элементы языка АСК

2.1 Элементами являются:

  • спецсимволы;
  • пустые символы;
  • комментарии;
  • номера и числа;
  • текстовая информация;
  • идентификаторы (обозначения).

2.2 Спецсимволы (видимые символы «не буквы» и «не цифры»)

Выполняют роль операторов в выражениях. Символы "_" и "$" не считаются спецсимволами и приравниваются к буквам.

2.3 Пустые символы

В качестве разделителей и для улучшения читаемости используются пробел, табуляция и перевод строки. Их мы называем пустыми символами. Они игнорируются рядом со спецсимволами и служат разделителями между идентификаторами.
Исключение: в начале строки перед меткой или мнемоникой команды пустые символы не допускаются.

2.4 Комментарии

Поясняющие записи, невидимые оператору во время выполнения. Поддерживаются два стиля:

  • «//» — всё до конца строки;
  • «/* ... */» — текст между этими маркерами.

Примеры:

*1=X1:1,X1:2   // комментарий линейный
*2=  /*X1:3,*/  X1:4   /* комментарий
многострочный */

2.5 Номер

Целое положительное число (32-битное), до 2147483647, записанное десятичными цифрами.

2.6 Число

Вещественное число — последовательность цифр с необязательной десятичной точкой.
Примеры: 1000, 1000.0, 150.3.
ВНИМАНИЕ: пустые символы вокруг точки недопустимы.
Также поддерживается научная форма: 150e3 (150000), 150e-3 (0.15).

2.7 Текстовая информация

Сопровождающие оператора сведения могут быть оформлены:

  • Фигурными скобками { ... } (может быть на нескольких строках);
    Пример: 30 СИ 100В, 100<МОм, П { метод накапливающего узла }
  • Кавычками " ... " или ' ... ' (на одной строке).
    При этом внутри нельзя использовать тот же ограничитель.
    Пример: 20 ПР *1-100 "Цепь 1" *101-200 'Блок "ABC". Цепь 2' *

Примечание: цветная подсветка поддерживается всегда для кавычек/апострофов и по опции для фигурных скобок.

2.8 Идентификаторы

Обозначения, вводимые пользователем, и предопределённые языком АСК (мнемоники команд, входы АСК и т.д.).
Различают простые и составные идентификаторы.

2.8.1 Простой идентификатор

Последовательность русских/латинских букв, цифр, "_" или "$".
Пример: X1, $name, _1, A_B.

2.8.2 Составной идентификатор

До четырёх простых идентификаторов, разделённых ":" (двоеточие), "/" (слеш) или "." (точка).
Примечание: пустые символы вокруг разделителей допустимы.
Примеры: "X1:1", "X1.1:1", "X1/1", "A1.A2/X1:1".

Длина — не более 79 знаков (пробелы внутри не считаются).

2.9 Уникальность идентификаторов

При разборе сначала игнорируется регистр, затем латинские буквы приводятся к «визуально соответствующим» русским (A, B, C, E, H, K, M, O, P, T, X и т.д.).
Примеры:

  • "b" ➔ "в"
  • "h" ➔ "н"
  • "k" ➔ "к"
  • "m" ➔ "м"
  • "t" ➔ "т"

Если идентификатор — или его последний компонент — число, то игнорируются лидирующие нули.
Примеры:

  • K01 ➔ K1
  • 002 ➔ 2
  • X001:001 ➔ X1:1
