2.1.10 Команда Run / Load — загрузка протокола выполнения

Горячая клавиша: Alt‑L (по умолчанию в ПО АСК)

Назначение

Загружает в ПО сохранённый ранее протокол выполнения (двоичный или текстовый файл) для последующего анализа. Двоичный протокол открывается в родном «окне протокола», текстовый — в режиме только для чтения с сохранением цветовых подсветок сообщений.

Как подготовить файл протокола

  1. Сделайте активным окно «Протокол» (где выводятся сообщения).
  2. Выполните команду File / Save (F2)
    → появится диалог «Сохранить протокол в файле».
  3. Выберите нужный формат (см. таблицу ниже) и укажите имя файла.
Опция в менюЧто получаем
«В формате двоичного файла» Полная копия протокола с мета‑информацией.
Загружается командой Run / Load или File / Open.
«В формате текстового файла» Плоский текст; полезен для пересылки по почте, просмотра в стороннем редакторе.

Загрузка протокола

При вызове Run / Load открывается системное окно «Загрузка файла‑протокола». Выберите файл — и ПО автоматически определит его тип:

  • Двоичный — откроется в «живом» протоколе; можно пролистывать, фильтровать сообщения, пользоваться поиском и т.д.
  • Текстовый — откроется в текстовом окне (режим «read‑only»), при этом поддерживается фирменная цветовая подсветка типов сообщений.

Альтернативный способ: команда File / Open — достаточно указать двоичный файл‑протокол; ПО распознает его и также откроет в родном формате.

Полезно знать

  • Работа с загруженным двоичным протоколом полностью идентична работе с протоколом, полученным «живьём» во время выполнения ПК или тестов.
  • Цветовая маркировка сообщений в текстовом окне помогает быстро находить ошибки, предупреждения и результаты измерений.
  • Храните архив двоичных протоколов вместе с соответствующими версиями ПК — это упростит ретроспективный анализ.
