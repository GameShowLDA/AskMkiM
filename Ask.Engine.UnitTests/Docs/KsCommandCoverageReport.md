# Покрытие команды КС

## Что покрыто

### 1. Исполнитель `KsCommandExecutor`

- Успешное измерение в обычном режиме.
- Успешное измерение с ключом `Д` и запись в документирование.
- Переключение в режим прозвонки по ключу `Б`.
- Ошибка измерения и публикация execution error.
- Ошибка измерения с одновременной записью в документирование.
- Открытая верхняя граница и расчёт целевого значения `firstValue + 10`.
- Вычитание сопротивления коммутатора в рабочем режиме.
- Прижим отрицательного результата к нулю.
- Поведение `idle`-режима без симуляции ошибок.
- Поведение `idle`-режима с принудительным браком.

### 2. Парсер `KcCommandParser`

- Корректный разбор команды с диапазоном и точками.
- Открытая верхняя граница.
- Открытая нижняя граница по фактическим метаданным команды.
- `KS001`: диапазон сопротивления отсутствует.
- `KS002`: параметры не распознаны.
- `KS003`: отсутствуют точки.
- `KS004`: пустое тело команды.
- `KS005`: точки указаны раньше диапазона.
- Недопустимый ключ для `КС`.
- Лишние хвостовые параметры.
- Специфичное правило `КС`: одиночная точка запрещена.
- Конфликты диапазона:
  - нижняя граница >= верхней;
  - нижняя граница > максимума;
  - верхняя граница > максимума.

### 3. Helper-валидация `ResistanceManager`

- Нижняя граница ниже минимально допустимой.
- Верхняя граница ниже минимально допустимой.

Эти два кейса вынесены отдельно, потому что текущий parser-синтаксис не пропускает отрицательные значения, а ветки в helper всё равно существуют и должны быть проверены.

## Что было исправлено по пути

- В `KsCommandExecutor` исправлена обработка открытой верхней границы.
- Для `КС` добавлен отсутствовавший код ошибки `KS005`.
- Парсер `КС` теперь отдаёт `KS004` для реально пустого тела.
- Парсер `КС` различает:
  - `KS001` — сопротивление не задано вообще;
  - `KS002` — параметры заданы, но не распознаны.
- Для `КС` исправлена привязка диапазонных ошибок к `KsErrors`, а не к ошибкам `ИЕ`.

## Важное замечание

В коде и документации есть расхождение по нижней границе `КС`:

- в HTML-справке указано `10 Ом`;
- в метаданных `MeasurementTypeCommand.KC` сейчас задан минимум `0`.

Тесты это не скрывают:

- parser-тест на открытую нижнюю границу зафиксирован по фактическому контракту кода;
- спорные ветки диапазона дополнительно закрыты helper-тестами.
