2. Разработка исходных текстов программ контроля

В состав ПО АСК входит собственный текстовый редактор (подробнее — раздел 11), предназначенный для создания и правки исходных программ контроля в кодировке CP‑866.

Подготовка рабочей директории

Предположим, что все ИПК будут храниться в C:\ipk. Откройте эту папку в проводнике или сделайте её текущей в командной строке.

Первый запуск ПО АСК

Запустите ПО АСК. На экране появится «подложка» серого цвета и верхняя строка меню. Красным выделена «горячая» клавиша каждого подменю — Alt + буква открывает нужный пункт. Меню также вызывается клавишей F10; перемещение по пунктам — стрелками. Подсказки выводятся в нижней строке экрана.

Создание нового файла

  1. Откройте меню File → New (Alt + F, затем N или нажмите красную букву прямо из главного меню).
  2. В появившемся окне введите имя файла, например first.pk. Редактор подставит полный путь C:\ipk\first.pk.
  3. Откроется новое окно для ввода текста. Верхняя рамка показывает номер окна и имя файла.
  4. Вводите текст: символы — с клавиатуры, новую строку — Enter, курсор — стрелками, удаление — Backspace/Delete.

Сохранение

Сохраните файл командой File → Save или горячей клавишей F2. Используйте её периодически, чтобы не потерять изменения.

Работа с несколькими окнами

Создайте второй файл second.pk. Его окно откроется уменьшенным. Увеличьте его командой Window → Zoom (переключает «максимальный ↔ заданный» размер). Для возврата к первому файлу примените Window → Window… и выберите нужный элемент списком. Так можно одновременно править несколько ИПК и копировать данные между ними.

Горячие клавиши и справка

  • F1 — контекстная помощь по выбранному пункту меню.
  • Alt + буква — быстрое открытие разделов меню.
  • Во всех меню справа от пункта показывается его «горячая» клавиша.

Открытие и закрытие файлов

File → Open открывает существующий файл (можно указывать только имя — путь дополняется текущей директорией). Window → Close закрывает окно; если файл изменён, редактор предложит сохранить правки.

Завершение работы

Команда File → eXit завершает сеанс. Перед выходом программа спросит о сохранении всех изменённых файлов и запишет конфигурацию (расположение окон, режимы и прочее), чтобы при следующем запуске восстановить состояние.

Подробное руководство по редактору — см. раздел 11.
