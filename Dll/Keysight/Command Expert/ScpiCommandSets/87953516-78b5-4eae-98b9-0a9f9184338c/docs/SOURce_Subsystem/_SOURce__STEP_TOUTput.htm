[SOURce:]STEP:TOUTput

Syntax | Description | Parameters | Remarks | Return Format | Examples

Syntax

[SOURce:]STEP:TOUTput <Bool>, (@<chanlist>)

[SOURce:]STEP:TOUTput? (@<chanlist>)

Description

This command specifies whether an output trigger signal is generated when a transient voltage or current step occurs. The state is either ON (1) or OFF (0). A trigger is generated when the state is True (ON).

Parameters

Name

Type

Range of Values

Default Value

<Bool>

Boolean

OFF | 0 | ON | 1

OFF

<chanlist>

Numeric

One or more channels.
(@2) - channel 2
(@1,4) - channels 1 and 4
(@1:3) - channels 1 through 3.

Required parameter

Return Format

<NR1>

The query command returns 0 if the trigger signal will NOT be generated (OFF) when a step occurs, and 1 if a trigger signal will be generated (ON) when a step occurs.

Examples

The following command sets the step trigger signal ON for channel 2.

STEP:TOUT ON,(@2)

The following query returns the step trigger signal state of channel 1.

STEP:TOUT? (@1)

Typical Response:  0
