[SOURce:]LIST:DWELl

Syntax | Description | Parameters | Remarks | Return Format | Examples

Syntax

[SOURce:]LIST:DWELl <time>{,<time>}, (@<chanlist>)

[SOURce:]LIST:DWELl? (@<chanlist>)

Description

This command specifies the dwell time for each list step. A comma-delimited list of up to 512 steps may be programmed. Dwell time is the time that the output will remain at a specific step. Dwell times can be programmed from 0 through 262.144 seconds with the following resolution:

Range in seconds

Resolution

0 - 0.262144

1 microsecond

0.262144 - 2.62144

10 microseconds

2.62144 - 26.2144

100 microseconds

26.2144 - 262.144

1 millisecond

List mode is only available with models N6761A/N6762A, or models that have Option 054 (High Speed Test Extensions).

Parameters

Name

Type

Range of Values

Default Value

<time>

NRf

0 - 262.144

1 step with a value of 0.001

<chanlist>

Numeric

One or more channels.
(@2) - channel 2
(@1,4) - channels 1 and 4
(@1:3) - channels 1 through 3.

Required parameter

Remarks

  • At the end of the dwell time, the output state of the unit depends upon the LIST:STEP program settings.

  • The order in which the dwell values are entered determines the sequence when the list executes.

  • To create a valid user-defined List, the Voltage, Current, BOST, EOST, and Dwell lists must either all be the same length, or have a length of 1, which is interpolated as having the same length as the maximum length list.

Return Format

<NR3>

The query command returns the programmed dwell time in the form +1.000000E+01. Multiple responses are separated by commas.

Examples

The following command programs a dwell list. The list contains 5 steps and is on channel 1.

LIST:DWEL 0.2,0.8,1.5,0.8,0.2,(@1)

The following query returns the dwell list settings of channel 1.

LIST:DWEL? (@1)

Typical Response:
+2.000000E-01,+8.000000E-01,+1.500000E+00,+8.000000E-01,+02.000000E-01
