[SOURce:]CURRent:PROTection:DELay:STARt

Syntax | Description | Parameters | Remarks | Return Format | Example

Syntax

[SOURce:]CURRent:PROTection:DELay:STARt <mode>, (@<chanlist>)

[SOURce:]CURRent:PROTection:DELay:STARt? (@<chanlist>)

Description

This command sets the over-current protection delay start event as follows:

SCHange

The over-current protection delay timer is started by a programmed settings change in the voltage, current, or output state. This is how the OUTPut:PROTection:DELay  command functioned prior to the availability of this command.

CCTRans

The over-current protection delay timer is started by any output transition to constant current mode.

This command is only available on model N6705A with firmware revision A.02.00 or later, and on N6700 mainframes with firmware revision C.03.00 or later.

Parameters

Name

Type

Range of Values

Default Value

<mode>

CPD

SCHange | CCTRans

SCHange

<chanlist>

Numeric

One or more channels.
(@2) - channel 2
(@1,4) - channels 1 and 4
(@1:3) - channels 1 through 3.

Required parameter

Return Format

<CRD>

The query command returns SCH or CCTR.

Example

The following selects CCTRans as the current protection delay mode.

CURR:PROT:DEL:STAR CCTR,(@1)

The following query returns the current protection delay mode.

CURR:PROT:DEL:STAR? (@2)

Typical Response:  SCH
