OUTPut:PROTection:DELay

Syntax | Description | Parameters | Remarks | Return Format | Example

Syntax

OUTPut:PROTection:DELay <delay>, (@<chanlist>)

OUTPut:PROTection:DELay? [MIN|MAX,] (@<chanlist>)

Description

This command sets the over-current protection delay. The over-current protection function will not be triggered on the specified output channel during the delay time. After the delay time has expired, the over-current protection function will be active. This prevents  momentary changes in output status from triggering the over-current protection function. Programmed values can range from 0 to 255 milliseconds in increments of 1 millisecond.

The operation of over-current protection is affected by the setting of the current protection delay start event, which is specified by [SOURce:]CURRent:PROTection:DELay:STARt.

Parameters

Name

Type

Range of Values

Default Value

<delay>

NRf+

0 - 0.255 | MIN | MAX

20 ms

<chanlist>

Numeric

One or more channels.
(@2) - channel 2
(@1,4) - channels 1 and 4
(@1:3) - channels 1 through 3.

Required parameter

Remarks

This command is the same as [SOURce:]CURRent:PROTection:DELay.

Return Format

<NR3>

The query command returns the programmed delay time. The reading is in the form +1.000000E+01.

Example

The following command sets a current protection delay of 0.2 seconds for channel 2.

OUTP:PROT:DEL 0.2,(@2)

The following query returns the maximum allowable current protection delay of channel 1.

OUTP:PROT:DEL? MAX,(@1)

Typical Response:  +2.550000E-01
