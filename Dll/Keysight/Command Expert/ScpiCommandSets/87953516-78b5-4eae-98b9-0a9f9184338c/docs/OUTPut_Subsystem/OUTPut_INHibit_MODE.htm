OUTPut:INHibit:MODE

Syntax | Description | Parameters | Remarks | Return Format | Example

Syntax

OUTPut:INHibit:MODE <mode>

OUTPut:INHibit:MODE?

Description

This command selects the mode of operation of the Inhibit input (INH). The inhibit function shuts down ALL output channels in response to an external signal on the Inhibit input. If an output channel has been turned off by OUTPut:STATe, the inhibit function does not affect the output channel while it is in the OFF state. The Inhibit mode setting is stored in non-volatile memory.

The following modes can be selected:

LATChing

Causes a logic-true transition on the Inhibit input to disable all outputs. The outputs remain disabled until the Inhibit input is returned to logic-false and the latched INH status bit is cleared by sending the OUTP:PROT:CLE command or a protection clear command from the front panel.

LIVE

Allows the enabled outputs to follow the state of the Inhibit input. When the Inhibit input is true, the outputs are disabled. When the Inhibit input is false, the outputs are re-enabled.

OFF

The Inhibit input is ignored.

Parameters

Name

Type

Range of Values

Default Value

<mode>

CPD

LATChing | LIVE | OFF

Required parameter

Return Format

<CRD>

The query command returns LATC, LIVE, or OFF.

Example

The following command sets the Inhibit Input to Live mode.

OUTP:INH:MODE LIVE

The following query returns the setting of the Inhibit Input.

OUTP:INH:MODE?

Typical Response:  OFF
