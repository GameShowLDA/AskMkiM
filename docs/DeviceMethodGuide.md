# Инструкция по добавлению нового метода в систему устройств

> ⚠️ Важно:\
> Эта система НЕ управляет реальными устройствами.\
> Она работает ТОЛЬКО с данными в базе (DTO → runtime объекты).
>
> Все методы --- это операции над данными БД.

------------------------------------------------------------------------

# Общая схема

IDeviceEngine\
→ DeviceEngine\
→ ICrudService\
→ CrudService\
→ DeviceRuntime
→ База данных

Плюс:

DeviceRuntime → Фасады (Racks, FastMeters и т.д.)

------------------------------------------------------------------------

# 1. Добавление метода в IDeviceEngine

Файл: IDeviceEngine.cs

Что сделать: - добавить сигнатуру метода

Пример:

Task`<int>`{=html} DeleteAllAsync`<TDevice>`{=html}(CancellationToken
cancellationToken = default) where TDevice : class, IDevice;

Что это значит: - метод будет доступен для всех типов устройств - он
работает с данными, а не с железом

Риск: если не добавить сюда --- всё ниже не скомпилируется

------------------------------------------------------------------------

# 2. Реализация в DeviceEngine

Файл: DeviceEngine.cs

Что сделать:

1.  Добавить публичный метод
2.  Сделать switch по типу устройства
3.  Вызвать нужный сервис

Пример логики:

if (requestedType == typeof(IRack)) return
DeleteAllInternalAsync\<TDevice, RackDto\>(...)

Что важно: - здесь происходит выбор нужной таблицы - никакой работы с
железом нет

Риск: можно удалить не ту таблицу, если ошибка в типе

------------------------------------------------------------------------

# 3. Внутренний метод (internal)

Что сделать:

-   вынести общую логику в отдельный метод

Пример:

private async Task`<int>`{=html} DeleteAllInternalAsync(...)

Что он делает: - вызывает CrudService - чистит кэш

Важно: обязательно очистить кэш:

\_cache.RemoveAll(typeof(TDevice));

Риск: если не очистить кэш --- будут старые данные

------------------------------------------------------------------------

# 4. Реализация в CrudService

Файл: CrudService.cs

Что сделать:

-   добавить метод, который реально работает с БД

Пример:

DELETE FROM Table

Что важно: - не загружать данные в память - делать 1 запрос в БД

Риск: если сделать через GetAll → RemoveRange: - будет медленно - будет
много памяти

------------------------------------------------------------------------

# 5. Добавление в DeviceRuntime

Файл: DeviceRuntime.cs

Что сделать:

-   просто прокинуть метод

Пример:

return Engine.DeleteAllAsync`<TDevice>`{=html}(...)

Важно: - здесь НЕТ логики - только прокси

Риск: если забыть --- метод не будет доступен снаружи

------------------------------------------------------------------------

# 6. Добавление в фасады устройств

Файлы: - Racks.cs - FastMeters.cs - и т.д.

Что сделать:

Пример:

public static Task`<int>`{=html} DeleteAllAsync() =\>
DeviceRuntime.DeleteAllAsync`<IRack>`{=html}();

Зачем: - чтобы не писать generic вручную - удобный API

Риск: без фасада пользоваться неудобно

------------------------------------------------------------------------

# 7. Очистка кэша

Когда нужно: - delete - update - batch операции

Что делать:

\_cache.Remove(...) или \_cache.RemoveAll(...)

Риск: старые данные в системе

------------------------------------------------------------------------

# 8. Проверка

Проверь:

1.  Метод есть в IDeviceEngine
2.  Есть реализация в DeviceEngine
3.  Есть метод в CrudService
4.  Есть прокси в DeviceRuntime
5.  Есть метод в фасаде
6.  Кэш очищается

------------------------------------------------------------------------

# Частые ошибки

❌ забыли добавить в интерфейс\
❌ забыли DeviceRuntime\
❌ забыли фасад\
❌ не почистили кэш\
❌ тянут всю таблицу в память\
❌ путают работу с БД и реальными устройствами
